# cp252-lab1
# Requirement document <br>
## Functional requirement <br>
1. ระบบการจองและยกเลิกที่จอด จองที่จอดล่วงหน้าผ่านแอป และกดยกเลิกการจองได้เอง
2. ระบบเช็คสถานะที่จอด แสดงจำนวนที่จอดว่างทั้งหมด และระบุตำแหน่งตามโซนต่างๆ
3. ระบบแอดมิน เจ้าหน้าที่สามารถเพิ่ม/ลด หรือแก้ไขจำนวนช่องจอดได้เอง
4. ระบบคิดเงินเองอัตโนมัติ คำนวณค่าจอดอัตโนมัติจากเวลาที่จอดจริงของรถแต่ละคัน
5. ระบบรีเซ็ทการจอง คืนสถานะเป็นที่ว่างทันทีหากหมดเวลาจองแล้วไม่มีรถเข้าจอด
6. ระบบค้นหาตำแหน่งรถ ระบุพิกัดหรือโซนที่จอดล่าสุด
7. ระบบแจ้งเตือน แจ้งเตือนเมื่อใกล้หมดเวลาจอง
8. ระบบชำระเงินออนไลน์ รองรับการจ่ายค่าจอดผ่าน QR Code, Mobile Banking หรือตัดบัตรเครดิต
9. ระบบสมาชิก เก็บประวัติการใช้งานและสะสมแต้ม
10. ระบบออกใบเสร็จดิจิทัล ส่งใบเสร็จค่าบริการเข้าอีเมล
## Non-Functional requirement <br>
1. ความรวดเร็วในการอัปเดต อัปเดตแบบเรียลไทม์
2. ความง่ายในการใช้งาน ยูสเซอร์สามารถเข้าถึงได้ง่าย เหมาะกับทุกช่วงวัย
3. ความแม่นยำของข้อมูล ป้องกันปัญหาลูกค้าไปแล้วไม่ตรงกับที่แอปบอก
4. รองรับการใช้งานจำนวนมาก ระบบต้องไม่ล่มถึงจะมีผู้ใช้งานเยอะ
5. ความปลอดภัยของข้อมูล ข้อมูลของผู้ใช้ทั้งหมดจะต้องเป็นความลับ

**ข้อกำหนดคุณสมบัติระบบบริหารจัดการที่จอดรถอัจฉริยะ (Smart Parking Management System)**

**1. ข้อกำหนดทั่วไปของระบบงานบริการผู้ใช้งาน (Mobile Application)**
1.1 สามารถใช้งานได้กับอุปกรณ์ระบบ IOS (iPhone/iPad) และระบบ Android OS เวอร์ชันปัจจุบันได้
1.2 มีการออกแบบส่วนติดต่อผู้ใช้งาน (User Interface) ที่ใช้งานง่าย ไม่ซับซ้อน เหมาะสมกับผู้ใช้งานทุกช่วงวัย (User-friendly)
1.3 ระบบต้องมีความเสถียร สามารถรองรับการเข้าใช้งานจำนวนมากได้พร้อมกันโดยระบบไม่ขัดข้อง (High Scalability)
1.4 ระบบต้องมีความรวดเร็วในการประมวลผลและอัปเดตข้อมูลแบบเรียลไทม์ (Real-time update) เพื่อให้ข้อมูลที่ปรากฏบนแอปพลิเคชันตรงกับสถานะจริงของพื้นที่จอดรถ และมีความถูกต้องแม่นยำสูง

**2. ระบบบริการจัดการสำหรับผู้ใช้งาน (Front-End)**
2.1 **ระบบตรวจสอบสถานะที่จอด:** สามารถแสดงจำนวนที่จอดว่างทั้งหมด และระบุตำแหน่งตามโซนต่างๆ ได้อย่างถูกต้อง
2.2 **ระบบค้นหาตำแหน่งรถ:** มีฟังก์ชันระบุพิกัดหรือโซนที่จอดล่าสุดของรถผู้ใช้งาน เพื่อความสะดวกในการค้นหา
2.3 **ระบบการจองและยกเลิก:** ผู้ใช้งานสามารถทำการจองที่จอดรถล่วงหน้าผ่านแอปพลิเคชัน และสามารถกดยกเลิกการจองได้ด้วยตนเอง
2.4 **ระบบรีเซ็ตสถานะอัตโนมัติ (Auto-Reset):** ระบบต้องสามารถคืนสถานะช่องจอดเป็น "ว่าง" ได้ทันทีแบบอัตโนมัติ หากผู้จองไม่นำรถเข้าจอดภายในเวลาที่กำหนด
2.5 **ระบบแจ้งเตือน (Notification):** มีระบบแจ้งเตือนไปยังอุปกรณ์ของผู้ใช้งานเมื่อใกล้หมดเวลาจอง หรือแจ้งเตือนสถานะที่เกี่ยวข้องกับการใช้งาน

**3. ระบบสมาชิกและการชำระเงิน (Membership & Payment)**
3.1 **ระบบสมาชิก:** มีระบบจัดเก็บประวัติการใช้งานของผู้ใช้ และรองรับการสะสมแต้ม (Point System) จากการใช้งาน
3.2 **ระบบคำนวณค่าบริการ:** มีระบบคำนวณค่าบริการที่จอดรถอัตโนมัติ โดยคำนวณจากเวลาที่จอดจริงของรถแต่ละคัน
3.3 **ช่องทางการชำระเงิน:** รองรับการชำระเงินออนไลน์ (Cashless Payment) ได้หลากหลายช่องทาง ได้แก่ QR Code, Mobile Banking และบัตรเครดิต
3.4 **ระบบออกใบเสร็จรับเงิน:** สามารถออกใบเสร็จรับเงินรูปแบบดิจิทัล (E-Receipt) และจัดส่งเข้าอีเมลของผู้ใช้งานได้

**4. ระบบบริหารจัดการสำหรับผู้ดูแลระบบ (Back-End)**
4.1 **ระบบจัดการพื้นที่จอด:** ผู้ดูแลระบบ (Administrator) สามารถบริหารจัดการ เพิ่ม ลด หรือแก้ไขจำนวนช่องจอด และกำหนดโซนที่จอดรถได้ เพื่อให้ข้อมูลสอดคล้องกับพื้นที่จริง
4.2 **ความปลอดภัยของข้อมูล:** ระบบต้องมีมาตรการรักษาความปลอดภัยของข้อมูล (Data Security) ในระดับสูง โดยข้อมูลส่วนตัวและข้อมูลทางการเงินของผู้ใช้งานทั้งหมดจะต้องถูกเก็บเป็นความลับและมีการเข้ารหัสข้อมูลที่ได้มาตรฐาน
